❗In MySQL, a FUNCTION cannot use OUT parameters.
Only PROCEDURE supports OUT parameters.

So your question is technically mixed:

FUNCTION → returns only one value
PROCEDURE → can return multiple values (OUT parameters)

DELIMITER $$

CREATE PROCEDURE Age_calc(

    IN dob DATE,
    OUT v_years INT,
    OUT v_months INT,
    OUT v_days INT

)

BEGIN

    DECLARE total_months INT;

    SET v_years =
    TIMESTAMPDIFF(YEAR, dob, CURDATE());
                                        -- calculate years

    SET total_months =
    TIMESTAMPDIFF(MONTH, dob, CURDATE());

    SET v_months =
    total_months % 12;
                                        -- calculate remaining months

    SET v_days =
    DAYDIFF(CURDATE()) - DAY(dob);
                                        -- calculate remaining days

END$$

DELIMITER ;

CALL Age_calc('2005-03-10', @y, @m, @d);

SELECT @y AS Years,
       @m AS Months,
       @d AS Days;

Theory
A Stored Function is a named PL/SQL block that returns a value using RETURN statement.

Features of Function

Conclusion
Thus, the Stored Function Age_calc was successfully created and executed to calculate age in years, months, and days.

pl/sql code:-
CREATE OR REPLACE FUNCTION Age_calc
(
    dob IN DATE,
    months OUT NUMBER,
    days OUT NUMBER
)

RETURN NUMBER

IS

    years NUMBER;

BEGIN

    years := FLOOR(MONTHS_BETWEEN(SYSDATE, dob) / 12);

    months := FLOOR(MOD(MONTHS_BETWEEN(SYSDATE, dob), 12));

    days := FLOOR(SYSDATE -
           ADD_MONTHS(dob,
           years * 12 + months));

    RETURN years;

END;
