Aim

To study and perform MongoDB CRUD operations using Employee collection.

Objective
To understand NoSQL database concepts.
To perform Insert, Update, Delete and Find operations.
To work with embedded documents and arrays.
To understand MongoDB query operators.

Theory

MongoDB is a NoSQL document-oriented database.
It stores data in the form of collections and documents instead of tables and rows.

MongoDB supports:

CRUD Operations
Create
Read
Update
Delete

Important MongoDB Functions:

insertMany() → Inserts multiple documents
find() → Retrieves documents
updateOne() → Updates single document
updateMany() → Updates multiple documents

MongoDB Operators Used:

$gt → Greater than
$inc → Increment value
$ne → Not equal
upsert:true → Inserts document if not found

Advantages of MongoDB:

Flexible schema
High performance
Easy handling of large data
Supports nested documents and arrays

Conclusion

Thus, MongoDB CRUD operations were successfully performed on Employee collection using different query operators, embedded documents and arrays.
